# AGENTS.md — PlayUp

Инструкции для AI-агентов, работающих с этим репозиторием (Codex, Cursor,
Gemini CLI, Copilot и др.). Открытый стандарт — https://agents.md.
Claude Code читает эти же правила через `CLAUDE.md`, который импортирует этот файл.

## Что это за репозиторий

`playup-docs` — **репозиторий документации и спецификации** продукта PlayUp.
Кода здесь нет. Это единый источник правды, по которому строятся отдельные
репозитории `playup-client-app` (Telegram mini app) и `playup-admin-app`
(внутренняя админка/CRM) поверх общей БД (PostgreSQL) через общий backend/API.

PlayUp — операционная система для клуба активной социальной жизни (спорт как
повод, а не услуга). MVP убирает ручной хаос (Google Sheets/Telegram), не
переавтоматизируя.

## Структура

Документы пронумерованы и читаются по порядку. Индекс с аннотациями — `00_README.md`,
полный список — `MANIFEST.md`.

- `00`–`12` — продуктовый проход (scope, архитектура, экраны, модель данных, план релизов)
- `13`–`24` — технический проход (API, схема БД, флоу, енумы, метрики, чек-листы, seed)
- `25` — дизайн-система (визуальный стиль и токены)

## Главная операционная петля

событие создано → клиент записался → команда видит запись → событие проведено →
отметили явку и оплату → система считает деньги/посещаемость/повторные визиты →
ушли напоминания и опросы. Всё, что петле не помогает, в MVP не входит.

## Жёсткие инварианты (не нарушать в любой спеке или коде)

- Три РАЗНЫХ факта не смешивать: Booking (записался) ≠ Attendance (пришёл) ≠ Payment (оплатил).
- Payment (факт оплаты участником) ≠ Transaction (движение денег в кассе).
- Revenue = `sum(transactions.income)`, НИКОГДА не `capacity * price`.
- `telegram_id` — главный внешний ключ, не username.
- Участники события только через Booking, не массивом в Event.
- draft-события клиенту не показываются.
- Источник правды по статусам/енумам — `19`, по API — `13`, по схеме БД — `14`,
  по метрикам — `21`, по флоу/стейт-машинам — `17`. Сверяться с ними, а не догадываться.

## Соглашения при работе с этим репозиторием

- Язык документов — русский. Сохранять тон и стиль существующих файлов.
- Новый документ продукта/спеки → следующий свободный номер `NN_name.md`.
- При добавлении/переименовании файла обновить индекс в `00_README.md` И `MANIFEST.md`.
- Не менять смысл продуктовых решений без явного запроса — при найденных
  нестыковках сначала сообщить, а не «чинить» молча.
- Коммитить только по просьбе. Сообщения коммитов — на русском, префикс `docs:`.

## Что НЕ делать

- Не добавлять сюда код приложений — он живёт в отдельных репозиториях.
- Не вводить фичи вне MVP (встроенная оплата, подписки, публичные профили,
  соцсеть, рейтинги, кабинет хоста/площадки, полноценная бухгалтерия).
