# 16. Client Screens Detailed Spec

## Назначение

Клиентская мини-апка должна дать пользователю быстрый путь: увидеть события, открыть карточку, записаться, увидеть свои записи и поправить профиль.

Главный принцип: сначала ценность, потом данные. Не показывать длинный онбординг до расписания.

## Навигация MVP

Три таба:

- Главная
- Расписание
- Профиль

Таб “Клуб” не входит в MVP.

## 1. Экран запуска

При открытии мини-апки клиент отправляет Telegram init data на backend. Backend создает или находит пользователя. После успеха открывается Главная.

Если авторизация не прошла, показать ошибку: “Не получилось открыть приложение. Попробуй зайти заново из Telegram.”

## 2. Главная

Цель: показать ближайшие записи пользователя.

Если есть записи:

- приветствие;
- ближайшее событие крупной карточкой;
- остальные будущие записи списком;
- кнопка “Открыть расписание”.

Карточка ближайшего события:

- название;
- дата;
- время;
- место;
- статус записи;
- короткая подсказка.

Если записей нет:

- “Ты пока не записан на события”;
- 2–3 ближайших открытых события;
- кнопка “Посмотреть расписание”.

## 3. Расписание

Цель: показать все доступные опубликованные события.

Карточка события:

- активность;
- название;
- дата;
- время;
- место;
- цена;
- свободные места;
- уровень;
- статус.

Статусы: доступно, почти заполнено, мест нет, вы записаны, отменено.

Фильтр MVP: все / футбол / падел / бег / йога / другое. Если фильтр тормозит запуск, его можно отложить.

## 4. Карточка события

Содержимое:

- название;
- активность;
- дата;
- время;
- длительность;
- площадка;
- адрес;
- цена;
- свободные места;
- уровень;
- хост;
- описание;
- что включено;
- важные заметки.

Состояния основной кнопки:

- “Записаться”;
- “Вы записаны”;
- “Отменить запись”;
- “Мест нет”;
- “Событие отменено”;
- “Событие прошло”.

## 5. Первый профильный шаг

Показывается только при первой записи, если профиль не заполнен.

Поля:

- имя;
- уровень;
- интересующий спорт;
- район / удобная локация;
- откуда узнал.

Заголовок: “Пара деталей, чтобы подобрать тебе нормальную игру”.

Кнопка: “Сохранить и записаться”.

Обязательные поля: имя, уровень, источник. Интересующий спорт можно предзаполнить активностью текущего события.

## 6. Подтверждение записи

После успешной записи показать:

- “Ты записан”;
- название события;
- дата;
- время;
- место;
- “Мы напомним перед событием”.

Кнопки: “На главную”, “Открыть расписание”.

## 7. Отмена записи

При клике “Отменить запись” показать подтверждение.

Текст: “Отменить запись на это событие?”

Кнопки: “Да, отменить”, “Оставить запись”.

После отмены booking становится `cancelled`, место освобождается, событие исчезает из ближайших записей.

## 8. Профиль

Содержимое:

- имя;
- Telegram username;
- уровень;
- интересующие активности;
- район;
- удобное время;
- источник привлечения;
- посещено событий;
- недель подряд;
- no-show;
- ближайшие записи.

Действия: редактировать профиль, открыть ближайшую запись.

## 9. Редактирование профиля

Поля:

- имя;
- уровень;
- активности;
- район;
- удобное время;
- телефон.

Источник привлечения лучше не редактировать в клиентском интерфейсе.

## 10. Опрос после события

Показывается после пуша или при открытии мини-апки, если есть активный опрос.

Содержимое:

- вопрос;
- оценка 1–5 или 1–10;
- текстовый комментарий;
- кнопка отправки.

После ответа — “Спасибо”.

## Состояния интерфейса

Нужны loading, error и empty states.

Примеры ошибок:

- “Не получилось загрузить расписание”;
- “Мест уже нет”;
- “Событие отменено”;
- “Не получилось записаться”.

## Не показываем клиенту

Не показывать внутренние заметки, расходы, чужие профили, список всех участников, внутренние статусы, ID и админские поля.

## Критерий готовности

Клиентский UI готов, если пользователь может открыть мини-апку, увидеть расписание, открыть событие, записаться, заполнить короткий профиль при первой записи, увидеть запись на Главной, отменить запись и изменить профиль.
